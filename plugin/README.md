# @appsent-co/react-native-watchos Expo plugin

An Expo Config Plugin that wires the `@appsent-co/react-native-watchos` Swift Package
(Hermes runtime + JSI host) into the watchOS target generated by
[`@bacons/apple-targets`](https://github.com/EvanBacon/expo-apple-targets)
during `expo prebuild`.

## Setup

### 1. Install

```sh
npx expo install @appsent-co/react-native-watchos @bacons/apple-targets
```

### 2. Create the watch target

```sh
npx create-target watch
```

This creates `targets/watch/` with `expo-target.config.json` containing
`{ "type": "watch" }` and a starter `index.swift`.

### 3. Configure plugins

In your `app.json` (order matters — `@bacons/apple-targets` must run first):

```json
{
  "expo": {
    "plugins": [
      "@bacons/apple-targets",
      ["@appsent-co/react-native-watchos", { "targetName": "watch" }]
    ]
  }
}
```

The `targetName` option defaults to `"watch"` (matching the directory name
under `targets/`). If you renamed the target directory, pass that name.

### 4. Drop in the POC ContentView (optional)

If you want a working "fetch a JS bundle and display console output" Watch
UI out of the box, copy `plugin/templates/ContentView.swift` into your
`targets/watch/` directory:

```sh
cp node_modules/@appsent-co/react-native-watchos/plugin/templates/ContentView.swift targets/watch/
```

It uses `ReactNativeWatchOSHost` from the SPM package to load a JS bundle
from `http://127.0.0.1:8000/hello.js` and renders `console.log` output in a
SwiftUI list.

### 5. Prebuild & run

```sh
npx expo prebuild -p ios --clean
# In another terminal:
cp node_modules/@appsent-co/react-native-watchos/plugin/templates/hello.js .
python3 -m http.server 8000
# Open Xcode, select the watch scheme, run.
```

## What the plugin does

During `npx expo prebuild`, after `@bacons/apple-targets` has created the
watch target in the pbxproj, this plugin:

1. Adds an `XCLocalSwiftPackageReference` pointing at the npm-installed
   `@appsent-co/react-native-watchos` package (relative path from `<project>/ios/`).
2. Adds an `XCSwiftPackageProductDependency` for the `ReactNativeWatchOS`
   library product.
3. Wires the product into the watch target's `Frameworks` build phase.
4. Relies on SPM's built-in handling to embed `Hermes.xcframework` (the
   dynamic JS engine) into the watch app.

The Hermes XCFramework is prebuilt and shipped inside the npm tarball at
`build/xcframework/Hermes.xcframework`. No `npm install` post-build steps
required.

## What the plugin does NOT do

- It does not create the watch target itself. `@bacons/apple-targets`
  handles that with `expo-target.config.json` — bundle ID, signing,
  deployment target, Info.plist, asset catalogs all flow through there.
- It does not bundle JS for the watch app — currently the Hermes runtime
  loads JS dynamically from a URL you provide. (Bundled JS support is a
  future enhancement.)

## Troubleshooting

**`Target "watch" not found in pbxproj.`** The `@bacons/apple-targets`
plugin didn't run before us. Make sure it's listed first in
`expo.plugins` and that `targets/watch/expo-target.config.json` exists.

**`Could not resolve '@appsent-co/react-native-watchos'`.** Run `npm install`. The
plugin uses `require.resolve` to find the package and the path is
included in the generated pbxproj as a relative path from `ios/`.

**SPM resolve fails at Xcode build time.** Check that
`build/xcframework/Hermes.xcframework` and
`build/xcframework/ReactNativeWatchOSCxx.xcframework` exist inside the
installed package. If you cloned the repo locally instead of installing
via npm, run `./scripts/build-xcframework.sh`.

**`No such module 'ReactNativeWatchOS'` in your `ContentView.swift`.**
SourceKit hasn't indexed the local SPM package yet — usually a Cmd-Shift-K
clean + rebuild fixes it. If it persists, verify the watch target's
"Frameworks, Libraries, and Embedded Content" lists `ReactNativeWatchOS`
in Xcode.
